description: Marketing Copy & Changelogs
---

1. **Phân tích Thay đổi (Changelog)**:
   - Xem lịch sử commit (`git log`) hoặc danh sách tính năng vừa hoàn thành.
   - Nhóm các thay đổi theo loại: Tính năng mới, Sửa lỗi, Cải thiện hiệu năng.

2. **Viết Changelog**:
   - Viết nội dung Changelog thân thiện với người dùng (không dùng ngôn ngữ kỹ thuật quá sâu).
   - Định dạng Markdown rõ ràng.

3. **Viết Marketing Copy (Nếu cần)**:
   - Nếu là tính năng lớn, viết đoạn giới thiệu ngắn (blurb) để thông báo (ví dụ: cho Newsletter, Social Media).
   - Tập trung vào lợi ích (Benefit) thay vì tính năng (Feature).

4. **Lưu trữ**:
   - Lưu vào `CHANGELOG.md` hoặc file tương ứng.
